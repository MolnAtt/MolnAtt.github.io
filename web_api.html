Statikus API-GET

Általános leírás

Valamely, a neten elérhető ingyenes és szabadon használható API segítségével készíts egy igényes statikus weboldalt! A honlapod rendelkezzen legalább egy index.html-lel, .css fájllal, .js fájllal. Ezek mindegyike legyen betöltve a html-fájlokba. Az általad választott API felhasználásával készítsd el a weboldalt.

A repositoryhoz tartozzon README.md fájl is, amelyben leírod, hogy mi volt nehéz, mi volt kevésbé nehéz, mire vagy nagyon büszke (esetleg mire vagy kevésbé büszke). Írd le bátran azt is, ha egy fejlesztést végül azért nem valósítottál meg, mert nem tudtad, hogyan kell. A házioldalon a mentor akkor is tud pár szóban iránymutatást adni, ha egyébként elfogadja a házi feladatod.

Figyelj arra, hogy a honlapon legyen valamilyen gomb is, amellyel ellenőrizhető, hogy valóban api-t használtál. (Pl. arra az esetre, ha az api minden lekérdezéskor más és más adatot ad! Ilyenkor pl a gomb kérje le újra az adatokat és generálja újra az objektumokat vagy frissítse az oldalt.)

Az elkészült weboldalt töltsd fel valamilyen statikus weboldalak hosztolására alkalmas oldalra! Egy kényelmes lehetőség, ha ezt még nem használod másra, a GitHub Pages. A link mögött egy tutorial van, lényegében ez úgy működik, hogy a GitHub fiókodban a speciálisan elnevezett, statikus weboldalt tartalmazó repositoryt képes a github hostolni.
